## Описание

REST API для управления подписками пользователей.

Приложение позволяет:

* создавать подписки;
* получать подписку по ID;
* получать список подписок пользователя;
* частично обновлять подписки (PATCH);
* удалять подписки;
* рассчитывать суммарную стоимость подписок за выбранный период с возможностью фильтрации по пользователю и названию подписки.

## Функциональность

### CRUD подписок

Подписка содержит:

* название;
* стоимость;
* UUID пользователя;
* дату начала;
* необязательную дату окончания.

Реализованы операции:

* Create
* Get by ID
* Get by User ID
* Partial Update (PATCH)
* Delete

---

### Расчет суммарной стоимости

Поддерживается расчет общей стоимости подписок за выбранный период.

Возможна фильтрация по:

* UUID пользователя;
* названию подписки.

## Запуск проекта

### 1. Клонировать репозиторий

```bash
git clone https://github.com/Lockok/efftest.git
cd efftest
```

### 2. Создать `.env`

Скопировать файл:

```bash
cp .env.example .env
```

или (Windows)

```powershell
copy .env.example .env
```

При необходимости изменить значения переменных окружения.

---

### 3. Запустить PostgreSQL

```bash
docker compose up -d
```

---

### 4. Применить миграции

```bash
make migrate-up
```

---

### 5. Запустить приложение

```bash
make app-run
```

После запуска API будет доступно по адресу:

```
http://localhost:8080
```

---

## Swagger

После запуска документация доступна по адресу:

```
http://localhost:8080/swagger/index.html
```

Через Swagger можно протестировать все реализованные endpoints.

## Примечания

* `end_date` является необязательным полем. (для `Create` тоже)
* Даты передаются в формате `YYYY-MM`.
* Для обновления подписки используется `PATCH`, поэтому можно передавать только изменяемые поля.
* При создании проекта использовалось логирование запросов и ошибок.
